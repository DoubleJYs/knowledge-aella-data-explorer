# CODEX_V2_TARGET.md

## V2 总目标

你正在改造本地项目：

`/Users/zhangjiyan/knowledge_land/aella-data-explorer`

该项目已经完成 v1 可运行骨架，并保留原 Aella 视图。V2 不以扩展 AI 能力和后端功能为主，而是专注于 **前端设计系统、页面交互、信息架构和可展示体验重构**。

V2 的产品定位：

> 面向高校和科研人员的审核型知识库可视化网站。它能把论文、笔记、课题资料等科研内容审核入库，并在统一设计系统下展示为可浏览、可搜索、可筛选、可点击查看详情的二维知识地图。

## V2 核心问题

v1 当前能跑通流程，但存在明显问题：

1. 前端缺少统一设计系统。
2. 页面像功能 Demo，不像正式产品。
3. 上传、审核、手动新增、地图页之间的用户路径不够清晰。
4. 信息层级、按钮优先级、空状态、加载状态、错误状态不统一。
5. 地图页交互不够像知识探索工具。
6. 原 Aella 视图需要保留，但不能继续作为主体验。

## V2 成功标准

V2 完成后，系统应该看起来像一个可展示的科研 SaaS 产品，而不是临时拼接页面。

必须达成：

1. 有统一 AppShell：左侧导航、顶部状态栏、主内容区。
2. 有统一 UI primitives：Button、Card、Badge、Input、Textarea、Select、Tabs、Table、EmptyState、LoadingState、ErrorState。
3. 首页从营销页改成科研工作台 Dashboard。
4. 上传页变成清晰的分步骤流程：选择知识库 → 上传文件 → 确认设置 → 进入待审核。
5. 审核页变成高效后台界面：左侧队列、中间预览、右侧表单或右侧详情。
6. 手动新增页有结构化表单和发布前预览。
7. 知识库管理页有卡片 / 表格两种清晰信息形态之一。
8. 地图页改成 Knowledge Explorer：顶部搜索工具栏、左侧筛选、中央地图、右侧详情面板。
9. 只展示 published 节点的规则不能破坏。
10. 原 Aella 视图保留在“实验视图 / Legacy Aella View”入口下，不作为主导航第一入口。
11. 无 AI Key、无外部 Agent、无 VPN 的情况下仍可运行。
12. TypeScript、Vite build、ruff、py_compile 和 Playwright 基础路径检查通过。

## V2 技术策略

优先小步前端重构，不重写后端。

默认建议：

1. 继续使用 React + TypeScript + Vite。
2. 继续复用现有 v1 API 和数据模型。
3. 地图的 D3 绘制逻辑只做外层容器和交互体验改造，不重写底层算法。
4. 可以引入轻量设计组件库，但必须先检查现有依赖和构建方式。
5. 推荐使用 Radix Themes 作为 V2 的快速设计基础；如果项目已存在 Tailwind / shadcn，再评估是否使用 shadcn/ui。
6. 每次修改前必须先阅读：
   - `docs/codex/CODEX_V2_FRONTEND_RULES.md`
   - `DESIGN.md`
   - `docs/project-structure-analysis.md`
   - 当前涉及页面和组件源码

## V2 不做事项

1. 不接入真实外部 AI / Agent API。
2. 不新增复杂权限系统。
3. 不重写后端业务流程。
4. 不重写 D3 地图核心绘制逻辑。
5. 不把未审核、驳回、草稿数据展示到前台地图。
6. 不删除原 Aella 视图。
7. 不一次性替换全项目样式。
8. 不引入大型 UI 框架或复杂动画库。
9. 不做移动端优先，先保证桌面端 1440px 和 1280px 体验。
10. 不把设计做成消费娱乐风格；保持高校科研、资料管理、数据探索的专业感。

## V2 推荐执行顺序

1. 只读审计：检查现有 v1 页面、组件、样式、路由、依赖。
2. 建立设计系统：更新 DESIGN.md，新增 UI primitives。
3. AppShell 改造：统一左侧导航和顶部栏。
4. 首页 Dashboard 改造。
5. 知识库管理页改造。
6. 上传页交互重构。
7. 后台审核页交互重构。
8. 手动新增页重构。
9. 地图页外壳和详情面板重构。
10. 原 Aella 视图归档为 Legacy 入口。
11. 统一空状态、加载状态、错误状态。
12. Playwright 路径走查和视觉验收。
