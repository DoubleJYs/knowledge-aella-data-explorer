# Codex V1 Target Mode 目标说明（3900 字符以内）

你正在改造本地项目：`/Users/zhangjiyan/knowledge_land/aella-data-explorer`。基座是已经拉取的 Aella Data Explorer。目标不是从零开发新系统，而是在现有 React + TypeScript + D3 前端和 FastAPI / SQLite 后端基础上，渐进式改造成 v1 可运行的“高校科研知识库可视化网站”。

v1 产品定位：面向高校教师、研究生、科研助理和课题组成员，把论文、笔记、项目资料等科研内容审核入库，并转化为可浏览、可搜索、可筛选、可点击查看详情的二维知识地图。

v1 核心模式是“审核型知识库”，不是“全自动 AI 平台”。现阶段没有自有微调大模型，外部 Agent / 大模型服务不能作为核心依赖。系统默认必须在无 AI Key、无外部 Agent、无 VPN 的情况下可运行。AI/Agent 自动分析只能作为可选增强，默认关闭；如果后续开启，也只能生成摘要、标签、分类等建议，必须经管理员审核后才可发布。

v1 必须跑通的主流程：管理员创建或使用默认知识库 → 用户上传科研资料 → 资料进入后台待审核队列 → 管理员查看上传内容并补全/修改标题、摘要、标签、资料类型、来源、年份等字段 → 管理员通过并发布 → 只有 `published` 状态的知识条目进入知识地图 → 用户在地图页浏览二维节点、搜索、筛选、点击节点查看右侧详情面板 → 可查看相似知识条目占位或 mock 数据。

v1 必做页面：1）首页：说明“科研资料审核入库并生成知识地图”，提供进入知识地图、上传资料、后台审核入口。2）知识库管理页：显示知识库卡片、已发布数量、待审核数量、打开地图、上传、审核入口。3）文档上传页：支持选择/拖拽文件，上传后状态为 `pending_review`，明确提示“审核通过后才会显示在地图”。4）后台审核页：左侧待审核队列，右侧审核表单，支持保存草稿、通过并发布、驳回。5）管理员手动新增条目页或弹窗：无需上传文件即可创建知识节点，方便演示和维护。6）知识地图页：复用 Aella 现有 D3 点云/聚类/缩放/点击逻辑，只展示 `published` 节点。7）右侧节点详情面板：展示标题、摘要、标签、聚类、来源、作者、年份、原文片段、相似条目。

工程原则：小步修改，先分析再改代码。每次任务开始必须先阅读 `docs/codex/CODEX_PROJECT_RULES.md` 和项目根目录 `DESIGN.md`。不要一次性重写项目、不要删除 Aella 原有核心可视化组件、不要引入复杂权限系统、不要接入真实外部 AI API、不要把未审核内容显示到前台地图。优先复用现有组件；字段从论文专用模型逐步抽象为 `KnowledgeItem`、`KnowledgeMapPoint`、`KnowledgeCluster`。

验收标准：项目能启动，原地图能力不被破坏；新增页面可访问；上传后进入待审核；后台可 mock 审核并发布；地图只显示 published 节点；无 AI 配置时系统仍可运行；README 或 docs 说明 v1 范围、运行方式和后续计划。
